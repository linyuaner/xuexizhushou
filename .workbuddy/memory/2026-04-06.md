## GitHub Actions CI/CD 工作流（学习助手项目）

完成了 CI/CD 自动化流水线搭建：

1. **.github/workflows/ci.yml** — 持续集成：
   - Job1 前端构建验证（npm ci + vite build + 上传 artifact）
   - Job2 后端验证（npm ci + Node.js 语法检查 + 启动测试）
   - Job3 前端单元测试（Vitest，--if-present 可选）
   - Job4 Docker 构建验证（docker build + 容器健康检查，无推送）
   - 触发：PR/push 到 main/develop

2. **.github/workflows/cd.yml** — 持续部署：
   - 构建多架构镜像（amd64+arm64）→ 推送到 GHCR + Docker Hub
   - SSH 部署到生产服务器（appleboy/ssh-action）
   - 支持 GitHub Environments 审批保护
   - 支持手动触发选择 production/staging 环境
   - paths 过滤：仅代码变更触发

3. **vitest.config.js** — Vitest 测试配置（jsdom 环境，覆盖率阈值 60%）

4. **src/utils/index.test.js** — 示例单元测试文件

5. **.gitignore** — 补充了缺失的 gitignore（含 node_modules、dist、.env、server/data）

6. **.github/ACTIONS_SETUP.md** — Secrets 配置说明文档

所需 GitHub Secrets：DOCKERHUB_USERNAME、DOCKERHUB_TOKEN、SSH_HOST、SSH_USER、SSH_PRIVATE_KEY、JWT_SECRET
